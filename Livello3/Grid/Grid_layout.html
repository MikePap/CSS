Grid Layout

Terminolgia e concetti

In generale

Il «Grid layout» si basa sul concetto di disporre un «container» formato da linee orizzontali e verticali (grid lines) che si intersecano per formare delle aree separate (grid areas) in cui verranno posizionati gli oggetti.

Un «grid container» viene creato impostando la proprietà display a grid o inline-grid ad un elemento. Questo crea un un contesto di formattazione di tipo griglia per il suo contenuto.
Tale contesto viene applicato soltanto ai primi elementi figli e non a elementi interni a quest'ultimi.
Il seguente è un semplice esempio: 

HTML    
<div class="box-grid">
    <div class="item"> </div>
</div>

CSS
div.box-grid {
    width: 300px;
    border: 1px solid;
    display: grid;
    grid-template-columns: 100px 100px 100px;
    grid-template-rows: 100px 100px;
}

.item {
    grid-column-start: 2;
    grid-column-end: 4;
    grid-row-start: 2;
    grid-row-end: 3;

    background-color: lightblue;
}

Invece di usare «grid-column-start» e «grid-column-end» possiamo dare «grid-column» e «grid-row» separate da uno slash (/).
Quindi rifacendoci all'esempio precedente potremmo dare: 

.item {
    grid-column: 2/4;
    grid-row: 2/3;
    background-color: lightgreen;
}

E' oltremodo possibile nominare le linee grid in modo da poterle referenziare più facilmente 

.box-grid {
    width: 300px; border: 1px solid;        
    display: grid;

    grid-template-columns: 100px [x1] 100px 100px [x2];
    grid-template-rows: 100px [y2] 100px [y2];
}
  
.item {
    grid-column-start: x1;
    grid-column-end: x2;
    grid-row-start: y2;
    grid-row-end: y2;    

    background-color: lightblue;
}

Linee e aree grid e grid track

Le linee grid sono linee orizzontali e verticali che formano la base della struttura del grid. Sono utilizzate per posizionare gli elementi all'interno della struttura.

Una grid area è formata da una o più «grid cell» e delimitata da quattro linee grid. Pertanto ci si può riferire ad essa dandogli un nome grazie alla proprietà grid-template-areas 

.box-grid {
    width: 450px; border: 1px solid;        
    display: grid;

    grid-template-columns: 150px 150px 150px;           tre colonne
    grid-template-rows: 150px 150px;                    due righe
    grid-template-areas: "a b b"
                         "a b b";
}

.item-1 {
    grid-area: a;
    background-color: lightgreen;
}

.item-2 {
    grid-area: b;
    background-color: lightblue;  
}

Il grid track è lo spazio che intercorre tra due linee grid orizzontali o verticali.

grid-template-areas

Permette di nominare le aree grid definite con «grid-template-columns» e «grid-template-rows» in modo da rendere più comprensibile la visualizzazione della struttura della griglia.
Abbiamo già visto qualche esempio quando si è parlato delle grid area

grid-template

È una scorciatoia per definire in una singola dichiarazione «grid-template-columns», «grid-template-rows» e «grid-template-areas».
L'ordine è importante; quindi prima le righe poi le colonne ed infine le aree.

grid-gap

Il «gap» sta ad indicare la distanza di separazione che intercorre tra le linee grid orizzontali o verticali.
Il grid-gap è una scorciatoia per definire in una singola dichiarazione grid-row-gap e grid-column-gap.

Quindi da quello che si deduce, «grid-row-gap» serve a definire la distanza tra le aree orizzontali e «grid-column-gap» tra quelle verticali.
Da tener presente che tali distanze di separazione non sono considerate all'inizio o alla fine del box grid.

La dichiarazione del «grid-gap» va scritta obbligatoriamente nel seguente ordine: 

grid-gap: <‘grid-row-gap’> <‘grid-column-gap’>

Se viene passato solo un valore si presuppone che valga per entrambi; quindi tale valore varrà sia per «grid-row-gap» che per «grid-column-gap».
Nel caso di due valori, bisogna separarli da uno spazio vuoto.

grid-auto-columns e grid-auto-rows

Quando un'elemento è piazzato in un container grid che non rientra nei confini definiti con «grid-template-columns» e «grid-template-rows» possiamo controllarlo attraverso queste due proprietà.
Vedere la sezione Grid esplicite ed implicite ed anche altri esempi qui

grid

È la scorciatoia per definire tutte le proprietà grid in una unica dichiarazione: quelle esplicite (grid-template-rows, grid-template-columns, grid-template-areas), quelle implicite (grid-auto-rows, grid-auto-columns, grid-auto-flow) e le gap (grid-column-gap and grid-row-gap).
Di seguito sono elencate tutte le proprietà ai loro valori iniziali (di default): 

grid-template-rows: none
grid-template-columns: none
grid-template-areas: none
grid-auto-rows: auto
grid-auto-columns: auto
grid-auto-flow: row
grid-column-gap: 0
grid-row-gap: 0

In una singola dichiarazione si possono anche solamente specificare le proprietà esplicite o quelle implicite; le restanti saranno impostate ai loro valori di default. 

.grid {
    grid: 50px 75px / auto-flow;
}
la dichiarazione precedente è equivalente alle seguenti:
.grid {
    grid-template-rows: 50px 75px;
    grid-template-columns: none;           cannot be set explicitly with this syntax form
    grid-template-areas: none;             cannot be set explicitly with this syntax form
    grid-auto-rows: auto;                  cannot be set explicitly with this syntax form
    grid-auto-columns: auto;
    grid-auto-flow: column;                can only set dense or not
    grid-column-gap: 0;                    cannot be set explicitly with this syntax form
    grid-row-gap: 0;                       cannot be set explicitly with this syntax form
}

Mentre la seguente:
.grid {
    grid: auto-flow dense / 30% 100px;
}

è equivalente alle seguenti:

    grid-template-columns: 30% 100px; 
    grid-template-areas: none;             cannot be set explicitly with this syntax form
    grid-auto-rows: auto; 
    grid-auto-columns: auto;               cannot be set explicitly with this syntax form
    grid-auto-flow: row dense;             can only set dense or not
    grid-column-gap: 0;                    cannot be set explicitly with this syntax form

La parola chiave repeat

La parola chiave «repeat» viene utilizzata per ripetere valori uguali. Un esempio è sempre meglio di tante parole 

CSS
#grid-repeat {
    display: grid;
    grid-template-columns: repeat(2, 50px 1fr) 100px;
    grid-gap: 5px;
    box-sizing: border-box;
    height: 200px;
    width: 100%;
    background-color: #8cffa0;
    padding: 10px;
}

#grid-repeat > div {
    background-color: #8ca0ff;
    padding: 5px;
}
    
HTML
<div id="grid-repeat">
    <div>
        This item is 50 pixels wide.
    </div>    
    <div>    
        Item with flexible width.
    </div>    
    <div>    
        This item is 50 pixels wide.
    </div>    
    <div>
        Item with flexible width.
    </div>
    <div>
        Inflexible item of 100 pixels width.
    </div>
</div>
This item is 50 pixels wide.
Item with flexible width.
This item is 50 pixels wide.
Item with flexible width.
Inflexible item of 100 pixels width.

Notando il «grid-template-columns», dove il «repeat» è stato definito, vediamo che il «repeat» ha creato due ripetizioni per il discorso colonne in cui ha definito una colonna di un valore fisso di 50px ed un'altra di valore flessibile usando «fr». Un'altra colonna è stata definita al di fuori del «repeat» che avrà valore 100px.
In definitiva si avranno cinque colonne di cui una sarà di 100px (quella più a destra), due saranno di 50px ed altre due che si divideranno lo spazio rimasto.
In altre parole, senza il «repeat», è come se fosse stato: 

grid-template-columns: 50px 1fr 50px 1fr 100px;

repeat con auto-fill, auto-fit e minmax

La parola chiave «repeat» può comprendere i parametri «auto-fill», «auto-fit» e la funzione «minmax()».

Grid esplicite ed implicite

Le grid esplicite sono quelle definite quando si impostano le proprietà «grid-template-rows», «grid-template-columns» e «grid-template-areas».
Ma se per esempio dovessimo definire una griglia con tre colonne e due righe possiamo oltremodo piazzare un suo elemento al di fuori di tale griglia. In questo caso il browser creerà una grid implicita per contenere tale elemento. 

.grid-container {
    display: grid;
    grid-template-columns: 150px 150px 150px;          tre colonne
    grid-template-rows: 150px 150px;                   due righe 
}
.item {
    grid-column: 5 / 6;                                parte dalla 5 per finire alla 6
    grid-row:  2 / 3;                                  parte dalla 2 per finire alla 3
}

Avremo qualcosa del genere:

Possiamo notare che le sei celle di colore grigio costituiscono la parte esplicita, definite con «grid-template-columns» e «grid-template-rows». Mentre avendo piazzato il nostro elemento nella quinta colonna (e seconda riga) il browser ha creato altre due colonne, oltre alle tre precedentemente definite; ha quindi creato una griglia implicita per poter visualizzare tale elemento.
Se avessimo piazzato il nostro elemento nella sesta colonna (6/7), sarebbero state create tre colonne e cosi via.

È anche possibile definire la grandezza delle celle implicite grazie alle proprietà grid-auto-columns e grid-auto-rows. Altrimenti il loro valore sarà «auto» ciò a dire che lo spazio di ogni cella sarà suddiviso equamente.

Osserviamo queto esempio in cui un grid container ha 10 elementi in esso:

.grid {
    display: grid;
    grid-template-columns: 150px 150px;
    grid-template-rows: 150px 150px;
    grid-auto-columns: 50px 75px;
    grid-auto-flow: column;
}
.item {
    grid-column: 8;
}

In questo caso abbiamo piazzato un nostro elemento nell'ottava colonna e quindi il browser ha generato altre sei colonne, implicite, oltre alle due esplicite. Però abbiamo impostato la proprietà «grid-auto-columns» dandogli due valori (50px e 75px).
Con questo vogliamo indicare al browser di creare le colonne che siano in serie una di 50px e l'altra di 75px ed inoltre dobbiamo indicargli anche la proprietà «grid-auto-flow» che sia appunto "column".

Piazzamento degli elementi nel grid container

Proprietà relative agli elementi del grid container

Le proprietà grid applicate al «container grid» sono usate per definire la struttura della griglia, mentre ci sono altre proprietà che sono applicabili agli elementi che permettono di posizionarli all'interno del «container grid». Sono i seguenti:

grid-column-start, grid-column-end e grid-row-start, grid-row-end
grid-column and grid-row
grid-area

grid-column-start, grid-column-end e grid-row-start, grid-row-end

Queste quattro proprietà permettono di definire la grandezza dell'elemento grid e dove deve essere posizionato all'interno della griglia.
In pratica definiscono le linee grid che formeranno i confini dell'area dell'elemento.

grid-column and grid-row

Sono scorciatoie che definiscono le linee di partenza e di chiusura.
«grid-row» è la scorciatoia per «grid-row-start» e «grid-row-end»; «grid-column» è la scorciatoia per «grid-column-start» e «grid-column-end».
I valori sono separati da uno slash ( / ). Il primo valore indica la linea di partenza ed il secondo la linea di chiusura.

grid-area

È una scorciatoia che imposta le linee che formano i confini dell'area dell'elemento (grid area).
Questa è una proprietà che conviene utilizzare in richiamo alla proprietà grid-template-areas definita nella definizione del «grid container»

Posizionamento degli elementi nel container grid

Il piazzamento di un elemento all'interno della griglia dipende dal grid position e dallo grid span.
Il «grid position» può essere specificato con le proprietà dei CSS (o altrimenti sistemato dagli algoritmi del browser) mentre il «grid span» determina quanti «grid track» l'elemento occupa. Di default è 1.
Ciò a dire che se abbiamo un grid-column-start = 1 e un grid-column-end = 3 vuol dire che il «grid span» è 2.

Ordinamento oggetti grid

L'ordinamento delle aree grid viene risolto in base all'ordine che gli elementi hanno nel documento HTML. Ad ogni modo è possibile cambiare tale disposizione HTML con le proprietà grid.

Per una migliore organizzazione e chiarezza per i designer è preferibile (nell'HTML) mettere in posizione superiore l'elemento <main>. Però per una migliore usabilità (specie per gli utenti che non possono usare il mouse) conviene che sia l'elemento <nav> ad essere in posizione superiore.
Grazie alle specifiche del «grid» possiamo ottenere entrambe le cose, nel senso che nell'HTML avremo l'elemento <main> in una posizione superiore (possibilmente sotto l'header) ma la visualizzazione finale vedrà l'elemento <nav> sotto l'header.

Sovrapposizione di elementi grid

Molteplici elementi possono essere posizionati nella stessa «grid cell». Li si può posizionare in modo che uno si sovrapponga all'altro. Per far ciò non c'è bisogno di specificare lo «z-index»; per default quello che sta più in basso nell'HTML, apparirà sopra gli altri.
In questo esempio si fa uso delle proprietà justify-self e align-self per creare un tipo di svrapposizione.

Posizionamento assoluto degli elementi del grid container

È possibile posizionare gli elementi del grid nel «grid container» in modo assoluto utilizzando la proprietà "position: relative". Invece l'utilizzo della proprietà "position: absolute" su un elemento è sconsigliata in quanto lo porta fuori dal flusso e non rientrerà nel layout del «grid container»

Il posizionamento assoluto degli elementi del grid può essere ottenuto con le proprietà grid-row e grid-column ma ciò non gli permette di riempire l'intera cella ma di estendersi fino a mostrare il suo contenuto.
top, right, bottom e left possono essere usati come di solito.

Allineamento del grid container e dei suoi elementi

Proprietà Allineamento

La specifica del grid utilizza i termini column-axis e row-axis per descrivere le direzioni degli allineamenti.
Il "column-axis" fa riferimento all'asse di blocco (senso verticale) mentre il "row-axis" fa riferimento all'asse inline (senso orizzontale)

Le proprietà che permettono di disporre il «grid container» e i suoi elementi sono le seguenti:

justify-content
align-content
justify-items
align-items
justify-self
align-self

justify-content

La proprietà «justify-content» ci permette di allineare il «grid container» nel senso orizzontale del «row-axis». Questo può tornarci utile quando la larghezza del «grid container» è maggiore del totale della larghezza degli elementi interni. O anche quando la larghezza non è definita e se il viewport è maggiore del totale della larghezza degli elementi interni.
I suoi possibili valori sono: 

justify-content: center | start | end | space-between | space-around | space-evenly

Vedi esempi

align-content

La proprietà «align-content» ci permette di allineare il «grid container» nel senso verticale del «column-axis». Questo può tornarci utile quando l'altezza del «grid container» è maggiore del totale dell'altezza degli elementi interni. I suoi possibili valori sono: 

align-content: center | start | end | space-between | space-around | space-evenly

Vedi esempi

justify-items

La proprietà «justify-items» ci permette di spostare il contenuto di un elemento nel senso orizzontale del «row-axis». Si comporta come la proprietà «text-align» ma agisce anche sulla larghezza dell'elemento, restringendola quando questa è superiore alla larghezza del contenuto.
I suoi possibili valori sono: 

justify-items: center | start | end | stretch

Vedi esempi

align-items

La proprietà «align-items» ci permette di spostare il contenuto di un elemento nel senso verticale del «column-axis».
I suoi possibili valori sono: 

align-items: center | start | end | stretch

Vedi esempi

justify-self

La proprietà «justify-self» si applica agli elementi del grid e non al grid container. Il comportamento risultante è simile a quello prodotto dal «justify-items».
Quindi la differenza sta nel fatto che in questo caso possiamo personalizzare il comportamento di ogni singolo elemento del «grid container» e non tutti insieme
I suoi possibili valori sono: 

justify-self: center | start | end | stretch

align-self

La proprietà «align-self» si applica agli elementi del grid e non al grid container. Il comportamento risultante è simile a quello prodotto da «align-items».
Quindi la differenza sta nel fatto che in questo caso possiamo personalizzare il comportamento di ogni singolo elemento del «grid container» e non tutti insieme
I suoi possibili valori sono:
